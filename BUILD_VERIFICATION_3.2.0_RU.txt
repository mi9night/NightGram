Обязательное развёртывание
1. Выполнить supabase/migration_call_history_web_push.sql.
2. Развернуть папку backend на Railway.
3. В backend выполнить npm run push:keys.
4. Добавить WEB_PUSH_PUBLIC_KEY, WEB_PUSH_PRIVATE_KEY и WEB_PUSH_SUBJECT в Railway.
5. Сохранить TURN-настройки и добавить HTTPS frontend в CLIENT_ORIGINS.

Ограничения проверки
- Реальная доставка Web Push зависит от HTTPS, браузера, разрешения пользователя и push-службы ОС.
- Реальные медиазвонки нужно проверить на физических Windows, Android и iPhone устройствах с настроенным TURN.
- На iPhone Web Push требует установленную PWA и поддерживаемую версию iOS/iPadOS.
