# NightGram 2.21.1 — двухэтапная защита через приложение-аутентификатор

NightGram больше не отправляет коды по email и не требует собственного домена, SMTP, Resend или SMS-провайдера.

## Как это работает

1. Пользователь устанавливает приложение-аутентификатор на телефон.
2. В NightGram открывает `Настройки → Безопасность → Двухэтапная защита`.
3. Вводит текущий пароль и сканирует QR-код.
4. Вводит шестизначный код из приложения.
5. Сохраняет одноразовые резервные коды.

Коды TOTP создаются на телефоне примерно каждые 30 секунд и работают без интернета.

## Рекомендуемые приложения

- **2FAS** — рекомендуемый вариант для большинства пользователей; Android и iPhone, открытый исходный код, резервное копирование: https://2fas.com/
- **Aegis** — приватный вариант для Android с зашифрованным хранилищем и резервными копиями: https://getaegis.app/
- **Microsoft Authenticator** — удобно для пользователей экосистемы Microsoft: https://www.microsoft.com/security/mobile-authenticator-app
- **Google Authenticator** — простой распространённый вариант для Android и iPhone: https://support.google.com/accounts/answer/1066447

Пользователю достаточно установить одно из этих приложений.

## Безопасность

- TOTP-секрет шифруется AES-256-GCM перед сохранением в Supabase.
- Ключ берётся из `TWO_FACTOR_ENCRYPTION_KEY`, а если переменная отсутствует — из существующего `JWT_REFRESH_SECRET` или `JWT_SECRET`.
- Принятый временный код нельзя использовать повторно.
- Резервные коды хранятся только в виде SHA-256 и работают один раз.
- QR-код создаётся локально в интерфейсе NightGram; секрет не отправляется стороннему QR-сервису.

Для отдельного ключа можно добавить на Railway:

```env
TWO_FACTOR_ENCRYPTION_KEY=очень-длинный-случайный-секрет
```

После включения 2FA эту переменную нельзя менять, иначе сохранённые секреты перестанут расшифровываться. Если отдельная переменная не задана, не меняйте используемые JWT-секреты без предварительного отключения 2FA у пользователей.

## Миграция

В Supabase SQL Editor выполните:

```text
supabase/migration_authenticator_two_factor.sql
```

Миграция безопасна как для чистой базы, так и после старой email-версии 2FA. Старые email-настройки 2FA будут отключены, чтобы пользователи могли войти и заново подключить приложение-аутентификатор.

После миграции разверните актуальную папку `backend` на Railway.
