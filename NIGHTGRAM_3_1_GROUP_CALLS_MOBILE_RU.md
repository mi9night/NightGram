# NightGram 3.1.0 — групповые звонки и мобильная стабилизация

## Что добавлено

- групповые аудио- и видеозвонки до 8 участников;
- отдельное WebRTC-соединение и отдельный медиапоток для каждого участника;
- динамическая сетка видео вместо объединения всех потоков в одно видео;
- подключение новых участников к уже идущему звонку;
- обычный участник может выйти, не завершая звонок для остальных;
- владелец звонка завершает конференцию для всех;
- отклонение одним участником не закрывает активный групповой звонок;
- 20-секундное окно восстановления после разрыва Socket.IO;
- повторная регистрация участника после reconnect;
- ICE restart для разорванных peer-соединений;
- защита от одновременных конфликтующих WebRTC-offer;
- отдельные состояния микрофона, камеры и демонстрации для каждого участника;
- демонстрация экрана передаётся всем участникам;
- переключение фронтальной и основной камеры на телефоне;
- смена микрофона и камеры прямо во время звонка;
- Wake Lock, чтобы поддерживаемый мобильный браузер не выключал экран;
- кнопка ручного запуска звука, если мобильный браузер заблокировал autoplay.

## Ограничение групповых звонков

В 3.1.0 используется mesh-архитектура: каждый клиент соединяется с каждым. Лимит установлен в 8 участников и проверяется одновременно frontend и backend.

Для конференций на десятки или сотни участников нужен отдельный SFU-сервер, например mediasoup, LiveKit или Janus. Это следующий самостоятельный архитектурный этап, а не безопасное расширение текущего mesh.

## Backend

Папку `backend` необходимо повторно развернуть на Railway. Новая SQL-миграция Supabase не требуется.

Для сложных сетей обязательно настройте TURN по `TURN_SETUP_RU.md`.
