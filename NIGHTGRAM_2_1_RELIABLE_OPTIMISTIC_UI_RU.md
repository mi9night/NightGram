# NightGram 2.1.0 — надёжный оптимистичный интерфейс

## Что изменено

- Сообщение появляется в чате сразу после нажатия «Отправить».
- При потере Socket.IO-соединения сообщение больше не исчезает: оно получает статус «Не отправлено».
- Для неотправленного сообщения доступна кнопка «Повторить».
- Через 12 секунд без подтверждения сообщение помечается как неподтверждённое вместо бесконечного статуса загрузки.
- Неотправленные сообщения сохраняются локально и восстанавливаются после перезапуска приложения.
- Серверная история объединяется с локальными неотправленными сообщениями без дублей.
- Клиентский `clientId` используется для сопоставления локального сообщения с серверным.
- Backend поддерживает идемпотентную повторную отправку: один и тот же `clientId` не создаёт второе сообщение.
- Быстрые повторные нажатия на лайк и сохранение последовательно синхронизируются с сервером.
- При ошибке лайк/сохранение возвращаются к последнему подтверждённому состоянию.

## Миграция Supabase

Для полной защиты от дублей примените файл:

`supabase/migration_message_client_ids.sql`

Backend остаётся совместимым со старой схемой, но без миграции защита от дублей после потерянного ACK действует не во всех сценариях.
