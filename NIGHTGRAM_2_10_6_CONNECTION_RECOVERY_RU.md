# NightGram 2.10.6 — контроль состояния сервера и восстановление связи

## Что изменилось

- NightGram отдельно проверяет доступность Railway API через `/api/health`.
- Проверка выполняется после входа, при возвращении приложения из фона, после восстановления интернета и периодически во время работы.
- При недоступности backend интерфейс переходит в понятный режим деградации и продолжает показывать сохранённую ленту, чаты и offline-outbox.
- Верхний индикатор различает отсутствие интернета, недоступность API и восстановление Socket.IO.
- В индикаторе появилась кнопка «Повторить», которая одновременно проверяет API и перезапускает соединение сообщений.
- В настройках Windows добавлен блок «Состояние сервера» с задержкой ответа, диагностическим кодом и ручным переподключением.
- После восстановления API NightGram повторно входит в Socket.IO-комнаты и запускает синхронизацию пропущенных данных.
- Health-check имеет короткий отдельный таймаут, поэтому не зависает на полном 25-секундном таймауте обычного API proxy.

## Совместимость

Обновление backend и SQL-миграция не требуются. Используется существующий маршрут Railway `/api/health`.
