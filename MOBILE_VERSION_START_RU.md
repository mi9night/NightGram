# Как запустить мобильную версию NightGram 3.3.1

Мобильная версия в архиве — устанавливаемая PWA для Android и iPhone/iPad. Отдельного APK/AAB и Xcode-проекта пока нет.

## Быстрый запуск в одной Wi-Fi сети

1. Установите Node.js 20 или новее на ПК.
2. Откройте папку `mobile-pwa-server`.
3. Запустите `START_MOBILE_SERVER.bat`.
4. Подключите телефон и ПК к одной Wi-Fi сети.
5. Узнайте локальный IP компьютера, например `192.168.1.20`.
6. Откройте на телефоне `http://192.168.1.20:3000`.

Так можно проверить интерфейс и сообщения. Камера, микрофон, установка PWA и Web Push обычно требуют HTTPS.

## Полноценный мобильный запуск

1. Опубликуйте frontend на Vercel или другом HTTPS-хостинге.
2. Разверните папку `backend` на Railway.
3. Если миграция 3.2 ещё не выполнялась, запустите `supabase/migration_call_history_web_push.sql`.
4. Добавьте домен frontend в Railway `CLIENT_ORIGINS`.
5. Настройте frontend/backend:

```env
BACKEND_API_URL=https://nightgram-production-0ceb.up.railway.app/api
NEXT_PUBLIC_SOCKET_URL=https://nightgram-production-0ceb.up.railway.app
```

6. Настройте TURN по `TURN_SETUP_RU.md`.
7. Настройте VAPID-ключи Web Push.
8. При необходимости установите длительность вызова до пропущенного:

```env
CALL_RING_TIMEOUT_SECONDS=60
```

9. Откройте HTTPS-домен на телефоне, войдите и перейдите: `Настройки → Уведомления → Включить`.

## Установка на главный экран

- Android/Chrome: меню → **Установить приложение**.
- iPhone/iPad/Safari: **Поделиться** → **На экран «Домой»**.

Для Web Push на iPhone PWA должна быть добавлена на домашний экран. Разрешение выдаётся только после действия пользователя.

## Звонки в 3.3.1

- личные и групповые звонки до 8 участников;
- демонстрация экрана на Windows и поддерживаемых desktop-браузерах;
- TURN для CGNAT, мобильных сетей и строгого NAT;
- входящий звонок через Socket.IO и Web Push;
- кнопки **Ответить** и **Отклонить** в поддерживаемых системных push-уведомлениях;
- восстановление активного звонка после быстрой перезагрузки приложения;
- восстановление после временной потери сети;
- автоматическое снижение качества видео при высоких потерях, задержке или недостаточном битрейте;
- индикатор качества связи внутри окна звонка;
- защита от двух одновременных звонков одного аккаунта.

При полном закрытии мобильной PWA входящий звонок сначала приходит как push. Полноценный нативный экран CallKit/Android Telecom потребует отдельного Android/iOS-приложения.

Для локального HTTPS-теста исходников используйте `START_MOBILE_PWA_HTTPS.bat`, но телефон должен доверять локальному сертификату.
