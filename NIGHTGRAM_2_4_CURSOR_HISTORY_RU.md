# NightGram 2.4.0 — быстрая история сообщений

## Что изменено

- Backend поддерживает cursor-пагинацию истории сообщений через `before` и `after`.
- При открытии чата загружаются только последние 80 сообщений.
- При прокрутке вверх более ранняя история подгружается порциями по 80 сообщений.
- После сна или восстановления сети клиент запрашивает только пропущенные сообщения, а не всю историю повторно.
- Поиск по загруженной переписке использует локальный триграммный индекс и не сканирует весь массив на каждое нажатие клавиши.
- При переходе к старому результату поиска в DOM остаётся ограниченное окно сообщений, а не вся переписка от результата до конца.
- Записи кэша объединяются: серия статусов доставки/прочтения создаёт одну актуальную запись вместо множества JSON-сериализаций.
- В локальном кэше сохраняются последние 500 подтверждённых сообщений и все ожидающие/ошибочные сообщения.
- Новое исходящее сообщение сразу обновляет превью диалога в списке чатов.
- Входящий диалог точечно обновляется и перемещается наверх своей группы без полной перезагрузки списка.

## Совместимость

Клиент 2.4.0 умеет работать со старым backend: если сервер возвращает обычный массив сообщений, чат продолжит работать, но глубокая подгрузка истории будет недоступна.

Для полной функциональности необходимо развернуть обновлённую папку `backend` на Railway. Миграция базы данных для cursor-пагинации не требуется.
