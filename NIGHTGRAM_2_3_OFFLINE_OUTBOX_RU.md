# NightGram 2.3.0 — офлайн-очередь сообщений

## Что изменилось

- Добавлен постоянный локальный outbox для исходящих сообщений.
- Сообщение без интернета получает статус «В очереди», а не теряется.
- Очередь хранится отдельно для каждого аккаунта и переживает перезапуск приложения.
- После восстановления Socket.IO сообщения отправляются автоматически из всех чатов, даже если активен другой раздел.
- Перед отправкой очередь переводит сообщения в состояние «Отправляется», а после ACK удаляет их из outbox.
- При серверной ошибке сообщение остаётся со статусом «Не отправлено» и доступно для ручного повтора.
- Добавлен глобальный индикатор: офлайн, переподключение, связь восстановлена, отправка очереди и ошибки outbox.
- Повторная отправка сохраняет стабильный clientId, поэтому сервер может безопасно устранять дубли.

## Сценарий проверки

1. Откройте любой чат при рабочем интернете.
2. Отключите интернет или Wi-Fi.
3. Отправьте несколько текстовых сообщений — рядом появится статус «В очереди».
4. Закройте и снова откройте NightGram: сообщения должны сохраниться.
5. Включите интернет.
6. После подключения Socket.IO сообщения должны автоматически перейти в отправленные.

## Важно

Для максимальной защиты от дублей на backend должна быть применена миграция `supabase/migration_message_client_ids.sql`, добавленная в NightGram 2.1.0.

Файлы и голосовые сообщения можно поставить в очередь только после того, как вложение уже было загружено и получило URL. Новую загрузку файла без интернета выполнить нельзя.
