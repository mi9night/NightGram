# NightGram 3.2.0 — Web Push и история звонков

## Добавлено

- Web Push для сообщений и входящих звонков;
- отдельная push-подписка для каждого браузера/устройства;
- краткоживущий входящий вызов восстанавливается после запуска приложения;
- Service Worker открывает нужный чат после нажатия уведомления;
- входящие звонки используют высокую срочность и постоянное уведомление;
- push учитывает главный переключатель уведомлений, категории чатов и тихие часы;
- часовой пояс сохраняется вместе с подпиской устройства;
- отложенные сообщения также отправляют Web Push;
- тест push из `Настройки → Уведомления`;
- отключение push отдельно на текущем устройстве;
- серверная история входящих, исходящих, завершённых, отклонённых и пропущенных звонков;
- раздел `Звонки` с длительностью, временем и повторным вызовом;
- безопасное восстановление активного входящего звонка после открытия PWA;
- исправлено получение TURN-конфигурации: backend теперь использует реальный `req.userId`.

## Web Push на Railway

В папке backend выполните:

```bash
npm install
npm run push:keys
```

Добавьте в Railway:

```env
WEB_PUSH_PUBLIC_KEY=...
WEB_PUSH_PRIVATE_KEY=...
WEB_PUSH_SUBJECT=mailto:admin@your-domain.example
```

Публичный и приватный VAPID-ключи должны быть стабильными. При их замене старые подписки браузеров потребуется включить заново.

## Ограничения

- Web Push требует HTTPS, кроме `localhost`;
- на iPhone push работает для установленной PWA на поддерживаемых версиях iOS/iPadOS;
- нажатие «Скрыть» закрывает системное уведомление, но не отклоняет звонок на сервере;
- для гарантированной доставки входящих через сложные сети всё равно нужен TURN;
- Web Push не заменяет будущие нативные FCM/APNs-клиенты.
